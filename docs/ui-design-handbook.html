01

文件資訊與產品定位

1.1 產品一句話

正式輔導前的低負荷情緒覺察與求助導引前哨站——非診斷、以使用者為中心，協助把「知道」變成「能照顧自己」。

1.2 核心能力對應

  • RULER：辨識／理解／標記／表達／調節情緒的引導路徑（打卡三步驟＋對話）。
  • CASEL SEL：自我覺察、自我管理、社會覺察、人際技巧、負責任決策（藝廊 SEL 曲集對應）。
  • 多模態回饋：心靈畫布（lib/mind-canvas.js）將情緒色票轉為視覺作品。

1.3 技術與檔案對應

層級主要檔案
殼層／樣式／流程emotion-app.html（單頁 App、CSS tokens、screens、tabs）
情緒／事件資料sel-data.v2.js（六大情緒類、70 色票、七類事件）
藝廊lib/emotion-gallery.js
背景音樂lib/app-music.js（App 頁 ~35–40%）；藝廊獨立曲集
畫布lib/mind-canvas.js
呼吸／危機／情境breathing.js、crisis-guard.js、scenario-play.js
02

品牌與體驗原則

溫暖

暖紙感背景（--bg #F4F1EE）、金色點綴（音樂鈕、關於 hero、夜間選取事件）、柔和圓角（--cr 22px）。

安全

SOS 常駐「你不是一個人」；危機／關注 modal；隱私鎖定（PIN／veil）；資料預設本機優先。

包容

身分分流、知情同意、低門檻選項（標籤優於長文）；文案避免指責與診斷口吻。

低負荷

一步一事；打卡分三步；快速回覆；語音輸入可選；夜間模式減輕刺激。

非診斷

介面與文案明確定位為引導工具，不取代專業諮商／醫療。

使用者主控

音樂開關、日夜切換、藝廊策展、支持對象需同意、SOS 可收合拖曳。

Splash 座右銘：「讓情緒被看見，讓安全被接住」·「溫暖 · 安全 · 包容」

3.3 情緒色（色彩心理學 · 六大類）

來源：sel-data.v2.js。每個情緒標籤有唯一色碼（共 70），供心靈畫布與 chip 使用。

高能量負面 · 憤怒

主色 #E53935 — 紅／朱＝喚醒與怒意。例：憤怒、暴躁、怨恨。

高能量負面 · 焦慮恐懼

主色 #FF7043 — 橙＝緊繃焦慮；紫＝恐懼未知。例：焦慮、害怕、窒息感。

低能量負面 · 悲傷耗竭

主色 #3A6EA5 — 藍／灰藍＝退縮與低能量。例：悲傷、孤獨、憂鬱。

複雜自我

主色 #7E57C2 — 紫／靛＝內省與羞恥。例：自我懷疑、羞愧、冒牌者感。

緩衝情緒

柔粉／茶褐／青綠。例：逞強、渴望被抱抱、想安靜一下。

正向情緒

主色 #FFD54F — 黃＝喜悅；綠＝踏實。例：開心、感恩、踏實。

3.6 元件基礎

  • .bp：主按鈕（實心 --acc）
  • .bg：次要／文字按鈕
  • .btn-canva-secondary：次要描邊鈕
  • .etag／.etag-on：標籤；夜間選取為金色魔法（.dark .etag.etag-on）
  • .sheet／.overlay：底部或置中面板
  • .app-music-btn：金色音符開關（關亦保持金色）
  • #sos-pill：常駐求援，柔和霓虹呼吸

3.7 圖示與 emoji

導覽以 SVG stroke 圖示為主；情緒／資源可用 emoji 作為溫暖識別（🤝 SOS、🎭 情境等），避免過度堆疊。藝廊工具列以文字＋狀態色為主。

04

資訊架構與導覽

4.1 主流程

Splash→ 登入／註冊→ 知情同意→ Onboard→ 主頁
Check-in 1 壓力→ 2 事件→ 3 情緒→ （高壓力則呼吸）→ 對話→ 結束／心靈畫布→ 日記／打卡／藝廊

4.2 主頁 Tabs（.bnav）

主頁陪伴卡、開始對話、情境練習、資源
日記自動紀錄網格；詳情含畫布與摘要
打卡牆連續天數、月曆熱力、趨勢、統計（最常觸發類型）
情緒藝廊全頁展場（另開 screen）
設定個人／陪伴者／支持對象（檢視↔編輯）
關於計畫背景、設計手冊、特色、團隊、隱私、AI 透明度

手機／平板：底部導覽。桌機 ≥1024：主頁改左側窄欄（同一組 DOM，僅 CSS 改排）。

05

逐頁介面規格

5.1 Splash

全螢幕品牌；輕觸進入。文案：座右銘＋「輕觸畫面進入系統」。SOS 於此頁隱藏（body.sos-splash）。

5.2 登入／註冊

  • 登入：「歡迎回來／你的私密情緒空間」；帳號密碼；Google；切換註冊。
  • 註冊：帳密＋Google；細節資料於 Onboard。
  • 容器：.auth-shell max-width 380–420px。

5.3 知情同意／Onboard

長文捲動同意；Onboard 分步填寫身分／學校／陪伴者名稱等。未成年身分觸發家長同意 UI。

5.4 Check-in（三步驟）

1/3 壓力標題「今天的你，好嗎？」；滑桿 1–10；返回＋繼續並排
2/3 事件七類客觀事件標籤；夜間選取金色魔法
3/3 情緒情緒輪盤／標籤模式；可語音＋文字反射

高壓力（≥7）或較多負向情緒 → 進入呼吸練習再對話。

5.5 呼吸（Box 4-4-4-4）

lib/breathing.js；環形視覺；可搭配身體掃描文案。語氣安撫、短句。

5.6 對話

  • 氣泡：AI／使用者；快速回覆（quick-replies.js）。
  • 危機：凍結輸入＋資源（crisis-guard.js）；關注級可選繼續。
  • SOS 在對話頁預設收合為圓鈕，避免遮擋輸入列。

5.7 對話結束／心靈畫布

.canvas-stage 固定 aspect-ratio 16/10；resize 依寬度重繪（不可拉成橢圓）。含深度摘要、日常小提醒、評分與回饋。

5.8 主頁

問候＋日期；.home-feature-grid 在 ≥768 為兩欄（陪伴卡｜情境卡）。文案例：「心裡有點小故事，都歡迎來聊聊。」右上：音樂＋日夜。

5.9 日記

空狀態溫和引導開始第一次對話；有資料時 .gallery-grid 2 欄（桌機 3 欄）。縮圖維持比例。

5.10 打卡牆

連續打卡橫幅、月曆（顏色反映情緒類）、近 10 次壓力趨勢、統計卡（最常觸發類型如「學業與課業」）。

5.11 情緒藝廊

  • 進場 portal（門扇／光線，偏慢儀式感）。
  • 近觀漫遊／遠距總覽／策展／紀錄片（≤7 張、ken burns、金游標魔法）。
  • 主題：柔光、雨日、清晨、白、米、黑（GALLERY_THEME_ORDER）。
  • 畫位 max 10；SEL 曲集音量為全站基準 100%。
  • 詳情：手機底部 sheet；平板／桌機右側分欄。

5.12 設定

區塊預設純文字展示；畫筆進入編輯；儲存後回展示。含 Gemini Key、PIN、學校資源等。

5.13 關於

金色 hero、TOC chips、計畫背景、本設計手冊、特色、團隊、補助、隱私、AI 透明度；右下回到頂部箭頭。

5.14 SOS

文案：「你不是一個人，如果需要幫助就在這裡。」可展開全國／校園資源；可收合、拖曳；異常座標自動校正回右下。

5.15 情境練習

I-message 說明 → 隨機情境角色扮演；與主對話分流。

06

響應式規範

原則：手機版（<768px）已完成，不重做、不大改。僅以媒體查詢／grid／flex 向上適配平板與桌機。
< 768 手機#app max-width 430px；底欄；單欄；既有互動
768–1023 平板#app ~760px；底欄加高；主頁兩欄；sheet 置中；藝廊詳情右側欄；畫位加大
≥ 1024 桌機#app ~1100px 置中留白；主頁左側導覽；日記 3 欄；展牆更寬；Hover 僅增強
≥ 1280藝廊可至 ~1360px；畫位 320px

驗收寬度：375 / 768 / 1024 / 1440。Canvas／藝廊圖在 resize 時依寬度重繪，維持比例。

07

互動與動效規格

  • 螢幕轉場：.38s cubic-bezier；exit-l／exit-r。
  • SOS 呼吸燈：sosNeonBreathe ~4.2s。
  • 主頁「開始對話」光環／環擴散。
  • 夜間事件選取：etagGoldMagic。
  • 藝廊進場、紀錄片 ken burns、游標金魔法。
  • 音樂：淡入淡出；App 頁音量約藝廊 35–40%（APP_MUSIC_VOLUME = 0.16）。

@media (prefers-reduced-motion: reduce)：關閉或簡化藝廊／ambient／portal 等動畫，保留可用性。

08

內容與合規

  • 非診斷：關於頁與知情同意反覆聲明。
  • 隱私：對話細節不未經同意外流；分析採去識別化。
  • 資源導引：1925、1995、1980、心快活、校園校安／諮商。
  • AI 透明度：ChatGPT／Claude／Gemini／Suno／Cursor／OpenRouter 等用途揭露。
  • 事件標籤：客觀中立名詞，不做結果論評價。
09

設計驗收清單

  • 375：主流程可走完；版面與改前一致；無溢位；SOS 可見
  • 768：兩欄主頁、加高底欄、藝廊詳情側欄合理
  • 1024：側欄導覽、內容置中留白、Hover 不取代點擊
  • 1440：藝廊／日記不無限拉寬；畫布不變形
  • 日／夜模式切換正常；音樂鈕金色；高壓力進呼吸
  • 危機語句觸發資源；關於手冊可預覽與列印 PDF
  • reduced-motion 下仍可操作關鍵流程
